## Part 1. Skills.md 반영 인사이트

### 1-A. 지표계산규칙.md — ETF 전용 지표 추가

**현재 상태**: ETF비교 유형에 샤프비율, 상관계수, 정규화가격만 존재
**보고서 근거**: "ETF 거래 시 중요한 괴리율(PE), NAV(순자산가치), 추적오차 등을 초보자도 이해하기 쉽게 시각화해야 한다" (보고서 §6)

| 추가 지표 | 산식 개요 | 적용 조건 |
|-----------|-----------|-----------|
| 괴리율(Premium/Discount) | `(시장가 - NAV) / NAV × 100` | ETF비교 유형 감지 시 |
| NAV 대비 가격 위치 | 현재가의 NAV 기준 상대 위치 | NAV 컬럼 존재 시 |
| 추적오차(Tracking Error) | 벤치마크 대비 일간 수익률 차이의 표준편차 | 벤치마크 컬럼 존재 시 |

**반영 우선순위**: 높음 — 평가 항목 "범용성 25점"에 직접 기여

---

### 1-B. 시각화매핑규칙.md — 경고 색상 조건부 렌더링 추가

**현재 상태**: ComposedChart, 게이지 차트 규칙은 이미 존재 (검증 완료). 단, **지표 임계값 기반 경고 색상 규칙**은 미비
**보고서 근거**: "1% 이상의 가격 오차가 발생했을 때 이를 경고 색상으로 표시" (§6)

| 추가 매핑 규칙 | 트리거 조건 | 시각 표현 |
|---------------|-------------|-----------|
| 괴리율 경고 색상 | 괴리율 ≥ 1% → 주황, ≥ 3% → 빨강 | 기존 차트/KPI에 조건부 fill/border 적용 |
| MDD 심각도 색상 | MDD ≥ -10% → 주황, ≥ -20% → 빨강 | KPI 카드 배경색 변화 |
| RSI 과매수/과매도 구간 | RSI > 70 or < 30 → 강조 | 차트 내 배경 영역 하이라이트 |

**반영 우선순위**: 중간 — 시각화 품질로 "대시보드 자동생성 25점" 기여

---

### 1-C. 인사이트생성규칙.md — 초보자 친화 메시지 변환 강화

**현재 상태**: 전문가/일반 이중톤 + 심각도 4단계 템플릿 존재
**보고서 근거**: "산식을 대시보드에 직접 노출하기보다 '현재 이 상품은 실제 가치보다 저렴하게 거래되고 있습니다'와 같은 메시지로 변환" (§6)

추가할 변환 템플릿 예시:
- 괴리율 < 0 → "현재 이 ETF는 실제 가치(NAV)보다 **저렴하게** 거래되고 있습니다"
- 괴리율 > 1% → "⚠️ 이 ETF는 실제 가치보다 **비싸게** 거래 중입니다. 매수 타이밍에 주의하세요"
- MDD 심화 시 → "최근 고점 대비 {n}% 하락했습니다. 과거 유사 구간에서 회복까지 평균 {d}일 소요되었습니다"
- 이동평균 크로스 → "단기 흐름이 장기 흐름을 상향 돌파했습니다 (골든크로스). 상승 전환 신호일 수 있습니다"

**반영 우선순위**: 높음 — 평가 항목 "실용성 10점" + 초보 사용자 접근성 직결

---

### 1-D. 시각화매핑규칙.md — 점진적 정보 공개 규칙 구체화

**현재 상태**: Level 1~3 점진적 공개 존재하나 전환 기준 불명확
**보고서 근거**: "기본 차트는 선 위주로 간결하게 보여주되, 버튼 하나로 이평선과 거래량이 포함된 상세 차트로 전환" (§6)

구체화 제안:
- **Level 1 (기본)**: 가격 선 차트 + KPI 카드 4개 이내 → 인지 부하 최소화
- **Level 2 (상세)**: 이동평균선, 거래량 바 오버레이, 비교 벤치마크 → 토글 1회
- **Level 3 (전문가)**: RSI, 볼린저밴드, 상관 히트맵, 이중축 → 토글 2회 또는 "분석 모드" 진입

**반영 우선순위**: 중간 — Skills.md 설계 완성도 향상

---

### 1-E. 데이터감지규칙.md — 데이터 신뢰성 검증 규칙 추가

**현재 상태**: 값 정규화, 인코딩 감지, 날짜 포맷 변환은 있으나 데이터 품질 검증 미약
**보고서 근거**: "데이터 신뢰성에 대한 보증 장치... 모든 지표가 최신 상태인지, 데이터 출처는 어디인지" (§2)

추가할 검증 규칙:
- 날짜 연속성 검사: 거래일 기준 결측 비율 > 10%이면 경고
- 이상치 플래그: 일간 수익률 ±20% 초과 시 데이터 오류 가능성 경고
- 데이터 최신성: 마지막 날짜가 30일 이상 과거이면 "오래된 데이터" 알림

**반영 우선순위**: 낮음 — 있으면 좋지만 핵심 평가 항목에 직접 영향 적음

## Part 2. 웹서비스 UI 개선 인사이트

### 2-A. 용어 순화 — 초보자 진입 장벽 제거

**보고서 근거**: "매수/매도 대신 구매하기/판매하기라는 일상 용어를 사용하여 심리적 거리감을 좁혔다" (§1)

현재 대시보드 적용 포인트:
- "변동성(Volatility)" → 뒤에 "(가격 흔들림 정도)" 병기
- "샤프비율" → "위험 대비 수익 효율" 보조 설명
- "MDD" → "최대 낙폭 (고점 대비 최대 손실)" 병기
- 인사이트 패널 텍스트를 일상어 우선으로 전환

**구현 난이도**: 낮음 — 텍스트 변경만으로 가능

---

### 2-B. KPI 카드 영역 최적화

**보고서 근거**: "상단에 현재 주가, 변동률 등 핵심 데이터만을 배치하여 인지 부하를 획기적으로 낮췄다" (§1)

개선 방향:
- KPI 카드를 최상단에 고정하되 **최대 4개**로 제한 (현재 데이터 유형별 자동 배정)
- 각 KPI에 상승/하락 색상(빨강/파랑) 명확 적용
- 숫자 크기를 키우고 라벨은 작게 — 숫자 우선 시각 위계

**구현 난이도**: 중간 — 기존 KPI 카드 컴포넌트 리팩토링

---

### 2-C. 차트 점진적 공개 UI 구현

**보고서 근거**: "기본 차트는 선 위주로 간결하게, 버튼 하나로 상세 차트 전환" (§6)

구현 방안:
- 각 차트 카드 우상단에 "상세 보기" 토글 버튼 추가
- 기본: 깔끔한 선 차트 (그리드선 최소화, 데이터 포인트 숨김)
- 상세: 이동평균선 오버레이 + 거래량 하단 바 + 툴팁 강화
- 전환 시 애니메이션으로 자연스러운 UX

**구현 난이도**: 중간~높음 — 차트 컴포넌트별 2-mode 렌더링 필요

---

### 2-D. 색상 시스템 통일

**보고서 근거**: "상승과 하락을 빨간색과 파란색으로 명확히 구분하여 시각적 이해를 돕고" (§1)

적용:
- 상승/양수: `text-red-500` (한국 증시 관례)
- 하락/음수: `text-blue-500`
- 중립/보합: `text-gray-400`
- 경고: `text-orange-500` (괴리율 1% 이상 등)
- 위험: `text-red-700` (MDD 심화 등)
- 모든 차트, KPI, 인사이트에 일관 적용

**구현 난이도**: 낮음 — Tailwind 유틸리티 클래스로 통일

---

### 2-E. 인사이트 패널 시각 강화

**보고서 근거**: "인포그래픽을 활용하여 메시지로 변환하여 전달하는 것이 토스증권 식의 초보자 중심 설계 철학" (§6)

개선 방향:
- 심각도별 좌측 컬러 바 (alert=빨강, warning=주황, info=파랑, positive=초록)
- 아이콘 활용 (⚠️ 🔺 🔻 ✅ 등 → Lucide 아이콘으로 대체)
- 각 인사이트 카드에 관련 차트로 스크롤 이동 링크
- 요약 배너: 전체 분석 결과를 1문장으로 최상단에 표시

**구현 난이도**: 중간 — 인사이트 컴포넌트 리디자인

---

### 2-F. 리스트 UI 효율화 (선택적)

**보고서 근거**: "카드 UI 대신 리스트 UI로 정보를 정렬하여 화면 공간을 효율적으로 활용" (§1)

적용 가능 영역:
- 데이터 편집 탭의 테이블 뷰 → 이미 리스트 형태일 가능성
- 인사이트 목록 → 카드보다 리스트가 스캔에 유리
- 다만 대시보드 차트 그리드는 카드 레이아웃 유지가 적절

**구현 난이도**: 낮음 — 인사이트 패널 레이아웃 조정

## Part 3. AI UI/UX 가이드 반영 인사이트

> 출처: `AI_Web_UIUX_Guide.md` (바이브코딩 & 하네스 엔지니어링 가이드)

### 3-A. DESIGN.md 작성 — 프로젝트에 현재 부재

**가이드 근거**: "프로젝트의 디자인 시스템을 마크다운 파일로 정리해 AI에게 제공... 코딩에서 CLAUDE.md를 쓰는 것과 같은 원리" (§1 원칙3)
**현재 상태**: InvestLens에 DESIGN.md 없음. CLAUDE.md에 코드 작성 규칙만 존재.

DESIGN.md에 포함할 항목:
- **브랜드 톤**: "복잡한 투자를 쉽게" — 토스증권 철학 차용
- **색상 팔레트**: 상승(red-500), 하락(blue-500), 경고(orange-500), 배경/텍스트 계열
- **타이포그래피**: 본문/헤딩 폰트, 크기 위계 (KPI 숫자 > 라벨)
- **컴포넌트 스타일**: KPI 카드, 차트 카드, 인사이트 카드의 기본 형태
- **레이아웃 원칙**: 12컬럼 그리드, 반응형 브레이크포인트, 최대 너비
- **참고 레퍼런스**: 토스증권 UI 스크린샷 URL

**반영 우선순위**: 높음 — 이후 모든 UI 작업의 일관성 기반. 바이브코딩 평가(15점)에 직접 기여.

---

### 3-B. 컴포넌트 라이브러리 하네스 적용

**가이드 근거**: "기존 Button, Card, Input 컴포넌트의 패턴을 유지해서 새 컴포넌트를 만들어줘. 임의로 새 스타일을 추가하지 말 것" (§3)
**현재 상태**: Tailwind CSS 사용 중이나, 공통 컴포넌트 체계가 명시적이지 않음

적용 방법:
- 현재 사용 중인 KPI 카드, 차트 래퍼, 인사이트 카드의 스타일을 추출 → DESIGN.md에 등록
- 이후 AI에게 새 컴포넌트 요청 시 "DESIGN.md의 카드 스타일 패턴을 따를 것" 명시
- Lint.md에 "새 컴포넌트 생성 시 DESIGN.md 참조 필수" Hard Rule 추가 고려

**반영 우선순위**: 중간 — DESIGN.md 작성 이후 자연스럽게 적용

---

### 3-C. 리뷰 게이트 체크리스트 도입

**가이드 근거**: "AI가 구현한 UI를 다음 기준으로 검토한 후 다음 단계로 진행" (§3)

InvestLens 맞춤 리뷰 체크리스트:
- □ DESIGN.md의 색상 팔레트만 사용했는가?
- □ Tailwind CSS만 사용했는가? (인라인 스타일 금지)
- □ 상승/하락 색상이 한국 관례(빨강/파랑)를 따르는가?
- □ KPI 카드가 4개 이내로 제한되어 있는가?
- □ 모바일 반응형이 깨지지 않는가?
- □ 차트 점진적 공개 Level이 Skills.md 규칙과 일치하는가?
- □ 인사이트 텍스트가 초보자 친화 톤인가?

**반영 우선순위**: 중간 — Lint.md에 통합 가능

---

### 3-D. 바이브코딩 워크플로우 적용

**가이드 근거**: "전체 레이아웃 초안 → 컴포넌트별 정교화 → 반응형 적용 반복 사이클" (§2, §4)

InvestLens에 적용할 사이클:
1. DESIGN.md 작성 → AI 컨텍스트 확보
2. 대시보드 전체 레이아웃을 DESIGN.md 기반으로 리패싱 요청
3. KPI 카드, 차트 영역, 인사이트 패널 각각 개별 정교화
4. 리뷰 게이트 통과 확인
5. Vercel 배포 → 실제 브라우저 검증 → 피드백 반복

**반영 우선순위**: 높음 — 작업 방식 자체의 개선. 바이브코딩 평가(15점)의 핵심.

---

### 3-E. 프롬프트 품질 개선 — 대시보드 특화

**가이드 근거**: "대시보드 페이지 → 사이드바 레이아웃, 카드 그리드, 데이터 시각화 라이브러리 명시" (§2)

대시보드 UI 요청 시 항상 포함할 정보:
- 사용 라이브러리: "Tailwind CSS + Recharts"
- 레이아웃: "12컬럼 그리드, KPI 최상단, 차트 2열 배치"
- 색상: "DESIGN.md 팔레트 적용, 상승=빨강 하락=파랑"
- 반응형: "모바일에서 1열, md 이상 2열"
- 컴포넌트 구조: "app/src/components/ 하위 기존 패턴 유지"

**반영 우선순위**: 높음 — 모든 UI 구현 요청에 즉시 적용 가능
