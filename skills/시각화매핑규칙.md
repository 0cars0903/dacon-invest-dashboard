# 시각화매핑규칙 (Visualization Mapping Rules)

> 데이터 유형과 계산된 지표를 기반으로 적절한 차트 타입을 자동 선택하고,
> 대시보드 레이아웃에 배치하는 규칙을 정의한다.
> 구현: Recharts 라이브러리 기준.

---

## 1. 차트 타입 카탈로그

| 차트 ID | 차트명 | Recharts 컴포넌트 | 최적 용도 |
|---------|--------|-------------------|-----------|
| `LINE` | 라인 차트 | `<LineChart>` | 시계열 추이, 가격 변동 |
| `AREA` | 영역 차트 | `<AreaChart>` | 누적 수익률, 추세 강조 |
| `BAR` | 바 차트 | `<BarChart>` | 기간별 비교, 범주 비교 |
| `COMPOSED` | 복합 차트 | `<ComposedChart>` | 가격 + 거래량 동시 |
| `PIE` | 파이 차트 | `<PieChart>` | 자산 배분, 비중 |
| `RADAR` | 레이더 차트 | `<RadarChart>` | 다차원 지표 비교 |
| `SCATTER` | 산점도 | `<ScatterChart>` | 위험-수익 관계 |
| `HEATMAP` | 히트맵 | 커스텀 (셀 그리드) | 상관계수 행렬 |
| `TREEMAP` | 트리맵 | `<Treemap>` | 섹터별 비중 |
| `CANDLESTICK` | 캔들스틱 | 커스텀 (SVG) | OHLC 가격 + 시장 심리 |
| `BUBBLE` | 버블 차트 | `<ScatterChart>` + Z축 | 위험-수익 + 규모(비중/시총) |
| `TABLE` | 테이블 | HTML Table | 요약 통계, 순위 |

## 2. 데이터 유형별 차트 매핑

### 2-1. STOCK_TS → 차트 배정

| 순서 | 지표 | 차트 타입 | 카드 크기 | 비고 |
|------|------|-----------|----------|------|
| 1a | 가격 추이 + 이동평균선 | `LINE` | FULL | 메인 차트, MA 20/60/120 오버레이 (OHLC 미존재 시) |
| 1b | OHLC 캔들스틱 + 거래량 | `CANDLESTICK` | FULL | OHLC 4개 컬럼 존재 시 1a 대신 표시, 하단 거래량 바 결합 |
| 2 | 누적 수익률 | `AREA` | FULL | 기준선(0%) 표시 |
| 3 | 거래량 | `BAR` | HALF | 가격 차트 하단 연동 (있을 때만, 1b와 중복 시 생략) |
| 4 | 일간 수익률 분포 | `BAR` (히스토그램) | HALF | 정규분포 오버레이 |
| 5 | 변동성 추이 (20일 롤링) | `LINE` | HALF | 조건: 데이터 ≥ 25일 |
| 6 | RSI | `LINE` | HALF | 30/70 기준선 표시, 조건부 |
| 7 | 볼린저 밴드 | `AREA` | FULL | 상단/하단 밴드 + 가격, 조건부 |
| 8 | 골든/데드크로스 이벤트 | `LINE` + 마커 | FULL | crossoverEvents 존재 시, 1a/1b에 이벤트 마커 오버레이 |
| 9 | 요약 통계 | `TABLE` | HALF | MDD, 변동성, 총수익률 등 |

**캔들스틱 차트 세부 규칙:**
- 양봉(종가>시가): `#10B981`(초록) 채움, 음봉(종가<시가): `#EF4444`(빨강) 채움
- 거래량 바: 양봉/음봉 색상 연동, 불투명도 50%
- 골든/데드크로스 마커: 삼각형(▲ 골든=초록, ▼ 데드=빨강), 차트 상단에 표시
- `logScaleRecommended=true`이면 Y축을 로그 스케일로 전환

## 3. 카드 레이아웃 시스템

### 그리드 기준
- 12컬럼 그리드 (Tailwind CSS `grid-cols-12`)
- 반응형: 모바일 1열, 태블릿 2열, 데스크탑 4열

### 카드 크기 정의
| 크기 | 데스크탑 | 태블릿 | 모바일 |
|------|---------|--------|--------|
| `FULL` | 12/12 (전체 너비) | 12/12 | 12/12 |
| `HALF` | 6/12 (반 너비) | 6/12 | 12/12 |
| `QUARTER` | 3/12 (1/4 너비) | 6/12 | 12/12 |

### 배치 규칙
1. **KPI 요약 카드** 최상단: 총 수익률, MDD, 변동성, 샤프비율 (QUARTER × 4)
2. **메인 차트** 다음: FULL 크기의 가격/누적수익률 차트
3. **보조 차트** 하단: HALF 크기 차트를 2열로 배치
4. **요약 테이블** 최하단

## 4. 스타일 규칙

### 색상 팔레트
```
메인 5색 (종목/자산 구분용):
  - #6366F1 (인디고) — 첫 번째 종목/메인
  - #06B6D4 (시안) — 두 번째 종목
  - #10B981 (에메랄드) — 세 번째 종목
  - #F59E0B (앰버) — 네 번째 종목
  - #EF4444 (레드) — 다섯 번째 종목

상승/하락:
  - 상승: #10B981 (초록)
  - 하락: #EF4444 (빨강)
  - 보합: #6B7280 (회색)

상관계수 히트맵:
  - -1.0: #3B82F6 (파랑)
  -  0.0: #F3F4F6 (회색)
  - +1.0: #EF4444 (빨강)
```

### 차트 공통 스타일
- 배경: `transparent` (카드 배경에 위임)
- 그리드 라인: `#E5E7EB` (밝은 회색, 점선)
- 축 텍스트: `#6B7280`, 12px
- 범례: 차트 상단, 수평 정렬
- 툴팁: 흰색 배경, 그림자, 정확한 수치 표시
- 애니메이션: 첫 렌더링 시 0.5초 페이드인

### 반응형 규칙
- 모바일: 범례 숨김, 축 라벨 축약, 터치 툴팁
- 데스크탑: 호버 툴팁, 전체 라벨

## 5. 계층적 필터링 및 점진적 정보 공개

### 5-1. 계층적 필터링 (Hierarchical Filtering)

| 필터 레벨 | 적용 범위 | 예시 |
|-----------|----------|------|
| 글로벌 필터 | 전체 대시보드의 모든 차트·KPI·인사이트에 적용 | 날짜 범위 선택, 종목 필터 |
| 컴포넌트 필터 | 특정 차트 카드 내부에서만 적용 | 개별 차트의 MA 기간 변경, 범례 토글 |

**글로벌 필터 구성:**
- 날짜 범위: 레인지 슬라이더 + 프리셋 (1M, 3M, 6M, YTD, 1Y, ALL)
- 종목 선택: 멀티셀렉트 드롭다운 (ETF_COMP, PORT_ALLOC)
- 데이터 빈도: 자동 감지값 표시 (수정 불가, 정보 표시용)

**필터 적용 방식:**
- 데이터 행 < 1,000: 실시간 필터링 (슬라이더 조작 즉시 반영)
- 데이터 행 ≥ 1,000: 배치 필터링 ("적용" 버튼 클릭 시 반영)

### 5-2. 점진적 정보 공개 (Progressive Disclosure)

사용자 숙련도에 따른 정보 깊이 조절:

```
Level 1 — 요약 뷰 (기본):
  - 요약 배너 + KPI 카드 4개 + 인사이트 패널 (상위 3개)
  - 모든 차트는 축소(collapsed) 상태, "더 보기" 토글 제공

Level 2 — 상세 뷰 ("더 보기" 클릭):
  - 전체 차트 카드 전개
  - 인사이트 전체 목록 (최대 10개)

Level 3 — 드릴다운 (차트 카드 클릭):
  - 확대 차트 + 수치 근거 + 원본 데이터 (인사이트생성규칙 §10 참조)
```

- `displayMode: 'simple'`일 때 기본 Level 1에서 시작
- `displayMode: 'expert'`일 때 기본 Level 2에서 시작
- 모바일: Level 1만 표시, 차트 카드는 스와이프 캐러셀

## 6. 조건부 렌더링 및 Fallback

### 기본 조건부 규칙

| 조건 | 처리 |
|------|------|
| 데이터 행 수 < 10 | 시계열 차트 대신 TABLE 우선 |
| 종목 수 > 10 | 상위 10개만 차트, 나머지는 TABLE |
| 날짜 범위 < 30일 | 장기 이동평균(60/120) 제외 |
| 수치 컬럼 1개만 존재 | 비교 차트(BAR 그룹, SCATTER) 제외 |
| 카테고리 값 > 15개 | PIE 대신 수평 BAR 사용 |

### Fallback 모드 (지표 부족 시)

지표계산규칙에서 `fallbackMode: true`가 반환된 경우, 전체 차트 구성을 단순화:

```
IF fallbackMode === true:
    차트 구성:
      1. TABLE — 전체 데이터 테이블 (FULL)
      2. TABLE — 기본 통계 요약: 평균, 중앙값, min, max (HALF)
      3. BAR — 컬럼별 값 분포 (수치 컬럼이 있을 때만) (HALF)
    KPI 카드:
      - 행 수 / 컬럼 수 / 결측률 / 데이터 유형
    인사이트:
      - "계산 가능한 투자 지표가 부족합니다. 기본 통계만 표시합니다." (info 레벨)
```

### 계산된 지표 수에 따른 점진적 구성

| 계산 가능 지표 수 | 차트 구성 |
|-------------------|----------|
| 0개 | Fallback 모드 (위 참조) |
| 1~2개 | KPI 카드 + TABLE + 해당 지표 차트 1개 |
| 3~5개 | KPI 카드 + 주요 차트 3개 + TABLE |
| 6개 이상 | 정상 모드 (유형별 전체 차트 배정) |

## 7. 차트 우선순위 오버라이드 (관리자용)

관리자(P2)가 차트 배치 순서를 커스터마이즈할 수 있도록, 각 데이터 유형별 차트 배정 테이블의 "순서" 컬럼은 **기본값**이며 오버라이드 가능:

```typescript
interface ChartConfig {
  overrides?: {
    dataType: string;           // 'STOCK_TS' | 'ETF_COMP' | ...
    chartOrder: string[];       // 차트 ID 배열 (표시 순서)
    hiddenCharts?: string[];    // 숨길 차트 ID
    customCharts?: {            // 커스텀 차트 등록 (P4 퀀트용)
      id: string;
      name: string;
      component: string;        // 컴포넌트명
      size: 'FULL' | 'HALF' | 'QUARTER';
      indicatorId: string;      // 필요 지표 ID
    }[];
  }[];
}
```

## 8. 인쇄/PDF 레이아웃 규칙

임원(P5), 자문사(P6)를 위한 인쇄 최적화:

| 규칙 | 내용 |
|------|------|
| 용지 | A4 세로 기본, 차트가 많으면 A4 가로 |
| 페이지 구성 | 1p: 요약 배너 + KPI 카드 4개 + 전체 요약문 |
|  | 2p~: 차트 (페이지당 최대 2개 FULL 또는 4개 HALF) |
|  | 마지막: 요약 테이블 + 인사이트 전체 목록 |
| 색상 | 화면과 동일 (컬러 인쇄 기준), 흑백 시 패턴 구분 |
| 헤더/푸터 | 날짜, 데이터 소스명, "InvestLens 자동 생성" |

---

**BUBBLE 차트 구현 참고:**
- Recharts `<ScatterChart>`에 Z축(원 크기)을 `<Scatter>` 데이터의 `z` 필드로 전달
- 최소 크기 = 8px, 최대 크기 = 40px (비중/거래대금 기준 선형 매핑)
- 각 버블에 라벨(종목명) 표시, 호버 시 상세 툴팁

**CANDLESTICK 차트 구현 참고:**
- Recharts에 캔들스틱 네이티브 지원 없음 → `<ComposedChart>` + 커스텀 `<Bar>` shape 사용
- 각 캔들: x=날짜, y=시가~종가 범위(body), whisker=저가~고가(shadow)
- 또는 SVG 커스텀 렌더러로 `<rect>` + `<line>` 조합
