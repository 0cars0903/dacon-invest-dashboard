# 지표계산규칙 (Indicator Calculation Rules)

> 데이터감지규칙에서 판별된 유형(STOCK_TS, ETF_COMP, PORT_ALLOC, CUSTOM)에 따라
> 자동으로 계산할 투자 지표와 그 계산 공식을 정의한다.

---

## 1. 공통 규칙

### 계산 우선순위
- 각 데이터 유형별 **필수 지표**를 먼저 계산
- 필수 지표 완료 후 **선택 지표**를 조건부 계산
- 결측값은 제외하고 계산, 결측 비율 > 30%이면 해당 지표 스킵 + 경고
- 모든 필수 지표가 계산 불가 → `fallbackMode: true` 설정, 기본 통계(평균/중앙값/min/max)만 제공

### 이상치 감지 (Anomaly Guard)
지표 계산 전, 원본 수익률에서 이상치를 감지하여 경고:

| 조건 | 판정 | 처리 |
|------|------|------|
| 일간 수익률 < -50% | 액면분할/병합 의심 | 경고 표시 + 해당 구간 마킹, 계산에는 포함 |
| 일간 수익률 > +100% | 액면분할/병합 의심 | 경고 표시 + 해당 구간 마킹, 계산에는 포함 |
| 가격이 0 또는 음수 | 데이터 오류 | 해당 행 제외 + 경고 |
| 연속 5일 이상 동일 가격 | 거래 정지 의심 | 경고 표시, 변동성 계산에서 제외 |

### 데이터 빈도 적응형 연환산 (Adaptive Annualization)
데이터감지규칙의 `frequency`와 `tradingDays`를 기반으로 연환산 계수를 자동 결정:

| 감지된 빈도 | 연환산 계수 (√N) | 설명 |
|-------------|-----------------|------|
| `minute` | √(tradingDays × 390) | 미국 기준 1일 390분 |
| `hourly` | √(tradingDays × 6.5) | 미국 기준 6.5시간 |
| `daily` | √(tradingDays ∥ 252) | tradingDays 없으면 252 기본값 |
| `weekly` | √52 | |
| `monthly` | √12 | |

- `tradingDays`가 제공되면 실제 거래일 수를 사용하여 정확도 향상
- 기본값(252)은 미국 시장 기준, 한국(249)도 오차 허용 범위

### 수치 포맷
| 지표 유형 | 소수점 | 단위 | 예시 |
|-----------|--------|------|------|
| 가격 | 2자리 | 원/$ | 72,450.00 |
| 수익률 | 2자리 | % | +12.34% |
| 비율 | 4자리 | 배수 | 1.2345 |
| 변동성 | 2자리 | % (연환산) | 25.67% |
| 상관계수 | 4자리 | -1 ~ +1 | 0.8523 |
| 비중 | 1자리 | % | 35.2% |

## 2. STOCK_TS (주가 시계열) 지표

### 필수 지표

#### 2-1. 일간 수익률 (Daily Return)
```
단순 수익률: R_t = (P_t - P_{t-1}) / P_{t-1}
로그 수익률: r_t = ln(P_t / P_{t-1})
```
- 입력: Close 가격 시계열 (useAdjusted=true이면 Adj Close 사용)
- 기본 출력: 단순 수익률 배열 (R_t)
- 로그 수익률 (r_t): `logScaleRecommended=true`이거나 데이터 기간 > 1년일 때 추가 계산
- 로그 수익률은 기간 수익률 합산이 정확하고, 변동성 계산에서 정규분포 가정에 더 적합
- 첫 번째 값은 NaN (전일 대비 불가)

#### 2-2. 누적 수익률 (Cumulative Return)
```
CR_t = (P_t / P_0) - 1
```
- P_0: 시작일 종가
- 출력: 기간 전체 누적 수익률 시계열

#### 2-3. 이동평균선 (Moving Average)
```
MA_n = (1/n) × Σ(P_{t-i}), i=0..n-1
```
- 기간은 데이터 빈도에 따라 자동 조정:

| 빈도 | 단기 | 중기 | 장기 | 비고 |
|------|------|------|------|------|
| `minute` | 5 | 15 | 30 | 분봉 기준 |
| `hourly` | 6 | 12 | 24 | 시간봉 기준 |
| `daily` | 20 | 60 | 120 | 일봉 기본값 |
| `weekly` | 4 | 13 | 26 | 주봉 기준 |
| `monthly` | 3 | 6 | 12 | 월봉 기준 |

- 데이터 길이 < n이면 해당 MA 스킵

#### 2-4. 변동성 (Volatility)
```
σ_annual = std(R) × √N    (N = 적응형 연환산 계수, 공통규칙 참조)
```
- 단기 MA 기간 롤링 변동성 계산 (일봉: 20일, 분봉: 5 등)
- 연환산: 공통규칙의 "데이터 빈도 적응형 연환산" 계수 적용

#### 2-5. 최대 낙폭 (Maximum Drawdown, MDD)
```
DD_t = (P_t - Peak_t) / Peak_t
MDD = min(DD_t)
```
- Peak_t: t시점까지의 최고가
- 출력: MDD 값 + 발생 기간(시작일~종료일)

### 선택 지표

#### 2-6. RSI (Relative Strength Index)
```
RS = avg(상승폭, 14일) / avg(하락폭, 14일)
RSI = 100 - (100 / (1 + RS))
```
- 기간: 14일
- 조건: 데이터 길이 ≥ 20일

#### 2-7. 볼린저 밴드 (Bollinger Bands)
```
상단 = MA_20 + 2 × std_20
하단 = MA_20 - 2 × std_20
```
- 조건: 데이터 길이 ≥ 25일

#### 2-8. 골든크로스 / 데드크로스 버퍼 조건 (Golden/Dead Cross with Buffer)
```
골든크로스 판정:
  1. MA_단기가 MA_장기를 상향 돌파 (기본: MA20 > MA60)
  2. 버퍼: MA_단기 > MA_장기 × 1.0001 (최소 0.01% 간격 유지)
  3. 거래량 확인: 돌파일의 거래량 > 최근 20일 평균 거래량 × 1.2 (20% 이상)

데드크로스 판정:
  1. MA_단기가 MA_장기를 하향 돌파 (MA20 < MA60)
  2. 버퍼: MA_단기 < MA_장기 × 0.9999
  3. 거래량 확인: 동일 조건
```
- 버퍼 조건으로 노이즈성 크로스오버 필터링
- 거래량 컬럼 미존재 시 거래량 조건 스킵 (가격 조건만 적용)
- 출력: `crossoverEvents: { date, type: 'golden'|'dead', volumeConfirmed: boolean }[]`
- 조건: 데이터 길이 ≥ MA_장기 기간 + 5일

## 3. ETF_COMP (ETF 비교) 지표

### 필수 지표

#### 3-0. 정규화 가격 (Base-100 Indexing)
```
NP_t(i) = (P_t(i) / P_0(i)) × 100
```
- 각 종목의 시작일 가격을 100으로 정규화하여 동일 스케일 비교
- 서로 다른 가격 범위(예: 삼성 70,000원 vs TIGER ETF 12,000원)를 직접 비교 가능
- 시각화에서 "정규화 가격 비교" 차트의 입력 데이터로 사용
- `logScaleRecommended=true`이면 로그 스케일 Y축 병행 표시

#### 3-1. 기간 수익률 (Period Return)
각 종목/ETF별 기간 수익률:
```
PR = (P_end / P_start) - 1
```
- 구간: 1개월 / 3개월 / 6개월 / YTD / 1년 / 전체
- 데이터 길이에 따라 계산 가능한 구간만 표시

#### 3-2. 샤프 비율 (Sharpe Ratio)
```
Sharpe = (R_avg - R_f) / σ
```
- R_f (무위험 수익률): 연 3.5% 기본값 (빈도별 환산: 0.035/N)
- R_avg: 일간 수익률 평균의 연환산
- σ: 일간 수익률 표준편차의 연환산

#### 3-3. 최대 낙폭 비교
각 종목/ETF별 MDD 계산 (STOCK_TS 2-5와 동일 공식)

#### 3-4. 상관계수 행렬 (Correlation Matrix)
```
ρ(A,B) = cov(R_A, R_B) / (σ_A × σ_B)
```
- 일간 수익률 기준 피어슨 상관계수
- 종목 수 ≥ 2일 때만 계산

### 선택 지표

#### 3-5. 베타 (Beta)
```
β = cov(R_i, R_m) / var(R_m)
```
- R_m: 첫 번째 종목(또는 가장 큰 시가총액)을 벤치마크로 사용
- 조건: 종목 수 ≥ 2

#### 3-6. 정보 비율 (Information Ratio)
```
IR = (R_i - R_benchmark) / Tracking Error
```
- 조건: 벤치마크 지정 가능 시

## 4. PORT_ALLOC (포트폴리오 배분) 지표

### 필수 지표

#### 4-1. 가중 수익률 (Weighted Return)
```
R_portfolio = Σ(w_i × R_i)
```
- w_i: 자산 i의 비중
- R_i: 자산 i의 수익률

#### 4-2. 자산 배분 비율
```
각 자산별: 비중(%), 카테고리별 합산 비중(%)
```
- 비중 합계 ≠ 100%이면 정규화 후 경고

#### 4-3. 포트폴리오 변동성
```
σ_p = √(w^T × Σ × w)
```
- Σ: 자산 간 공분산 행렬
- 수익률 데이터 없으면 → 개별 자산 수익률 × 비중으로 근사

#### 4-4. 카테고리별 집중도
```
HHI = Σ(s_i²)
```
- s_i: 카테고리 i의 비중 (0~1)
- HHI > 0.25: "집중됨" / HHI < 0.15: "분산됨"

### 선택 지표

#### 4-5. 리밸런싱 시뮬레이션
- 조건: 날짜 + 가격 데이터 존재 시
- 분기별 리밸런싱 vs 유지 수익률 비교

## 5. CUSTOM 유형 지표

| 서브타입 | 계산 지표 |
|---------|----------|
| `CUSTOM_TS` | 추세(이동평균), 변동성, 기간별 변화율 |
| `CUSTOM_CAT` | 범주별 합계·평균·최대·최소, 비중 |
| `CUSTOM_NUM` | 기술통계(평균, 중앙값, 표준편차), 상관관계 |
| `CUSTOM_RAW` | 기본 통계(행 수, 컬럼별 요약)만 제공 |

### 5-1. 다중요소 스코어링 (Multi-Factor Score) — 선택

ETF_COMP 또는 CUSTOM_NUM에서 수치 컬럼이 3개 이상일 때 자동 적용:

```
각 수치 컬럼을 Min-Max 정규화 (0~1 범위):
  norm_j = (x_j - min_j) / (max_j - min_j)

종합 점수 (균등 가중):
  Score_i = (1/K) × Σ norm_j(i),  j=1..K
```

- K: 수치 컬럼 수
- 역방향 지표 (변동성, MDD 등 낮을수록 좋은 지표): `1 - norm_j`로 역변환
- 역방향 판정 기준: 컬럼명에 `risk`, `volatility`, `변동성`, `mdd`, `drawdown` 포함 시
- 출력: `multiFactorScores: { name: string, score: number, rank: number }[]`

## 6. 출력 스키마

```typescript
interface CalculationResult {
  dataType: string;
  fallbackMode: boolean;       // true이면 필수 지표 계산 불가, 기본 통계만 제공
  anomalies: {                 // 이상치 감지 결과
    date: string;
    type: 'split_suspect' | 'price_error' | 'halt_suspect';
    value: number;
    description: string;
  }[];
  indicators: {
    id: string;           // 예: 'cumulative_return'
    name: string;         // 예: '누적 수익률'
    category: 'required' | 'optional';
    value: number | number[] | Record<string, number>;
    unit: '%' | '$' | '원' | 'ratio' | 'index';
    timeSeries?: {        // 시계열 지표인 경우
      dates: string[];
      values: number[];
    };
    metadata?: {
      period?: string;    // 예: '20일'
      benchmark?: string;
      nullCount: number;
    };
  }[];
  crossoverEvents?: {            // 골든/데드크로스 이벤트
    date: string;
    type: 'golden' | 'dead';
    volumeConfirmed: boolean;
  }[];
  normalizedPrices?: {            // Base-100 정규화 가격 (ETF_COMP)
    ticker: string;
    dates: string[];
    values: number[];
  }[];
  multiFactorScores?: {           // 다중요소 종합점수
    name: string;
    score: number;                // 0~1
    rank: number;
  }[];
  summary: {
    totalReturn: number;
    annualizedReturn: number;
    volatility: number;
    sharpeRatio?: number;
    mdd: number;
  };
}
```

## 7. ETF 전용 추가 지표 (v2.2 신규)

> ETF_COMP 유형 감지 시 아래 지표를 추가 계산한다.
> NAV 컬럼이 없는 경우 해당 지표는 SKIP하고 경고를 표시한다.

### 7-1. 괴리율 (Premium / Discount Rate)

```
괴리율(%) = (시장가 - NAV) / NAV × 100
```

| 조건 | 적용 |
|------|------|
| NAV 컬럼 필수 | `nav`, `NAV`, `순자산가치` 컬럼명 매칭 |
| 시장가 컬럼 | `price`, `close`, `종가` 우선 사용 |
| 단위 | 퍼센트(%), 소수 2자리 반올림 |
| 기호 | 양수(+) = 프리미엄, 음수(-) = 디스카운트 |

**이상 임계값:**
- `|괴리율| < 0.5%` → 정상
- `0.5% ≤ |괴리율| < 1%` → 주의 (info)
- `1% ≤ |괴리율| < 3%` → 경고 (warning)
- `|괴리율| ≥ 3%` → 위험 (alert)

---

### 7-2. NAV 대비 현재가 위치

```
NAV 위치(%) = 괴리율과 동일 (표현 방식만 다름)
NAV 위치 레이블:
  - 괴리율 > 0: "NAV 대비 {n}% 비싸게 거래 중"
  - 괴리율 < 0: "NAV 대비 {|n|}% 저렴하게 거래 중"
  - 괴리율 = 0: "NAV와 동일 가격에 거래 중"
```

**게이지 차트 입력값으로 활용:**
- center value = 괴리율
- range = [-5%, +5%] (범위 초과 시 min/max에 클램핑)

---

### 7-3. 추적오차 (Tracking Error)

```
일간 추적오차(i) = ETF 일간수익률(i) - 벤치마크 일간수익률(i)
연환산 추적오차 = std(추적오차 시계열) × √주기_계수
  - 일간: √252, 주간: √52, 월간: √12
```

| 조건 | 적용 |
|------|------|
| 벤치마크 컬럼 필수 | `benchmark`, `index`, `기준지수` 컬럼명 매칭 |
| 최소 데이터 | 20거래일 이상 필요, 미만 시 SKIP |
| 단위 | 퍼센트(%), 소수 2자리 |

**이상 임계값:**
- `추적오차 < 0.5%` → 우수 (positive)
- `0.5% ≤ 추적오차 < 1%` → 보통 (info)
- `1% ≤ 추적오차 < 2%` → 주의 (warning)
- `추적오차 ≥ 2%` → 위험 (alert)

---

### 7-4. 출력 스키마 확장 (ETF 전용 지표)

```typescript
// CalculationResult에 추가
etfMetrics?: {
  premiumDiscount: {           // 괴리율
    current: number;           // 최신값(%)
    history: { date: string; value: number }[];
    level: 'normal' | 'info' | 'warning' | 'alert';
    label: string;             // "NAV 대비 X% 저렴하게 거래 중"
  } | null;
  trackingError: {             // 추적오차
    annualized: number;        // 연환산(%)
    level: 'positive' | 'info' | 'warning' | 'alert';
  } | null;
  navGauge: {                  // 게이지 차트 입력
    value: number;             // 괴리율(%)
    min: number;               // -5
    max: number;               // +5
    clamped: boolean;
  } | null;
};
```
