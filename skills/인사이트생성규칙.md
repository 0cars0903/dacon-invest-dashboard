# 인사이트생성규칙 (Insight Generation Rules)

> 계산된 투자 지표를 사람이 읽을 수 있는 해석 문구(인사이트)로 변환한다.
> 각 인사이트에는 심각도 레벨, 우선순위, 표시 위치가 부여된다.

---

## 1. 인사이트 구조

### 1-1. 단일 인사이트 스키마

```typescript
interface Insight {
  id: string;                    // 고유 식별자
  level: 'alert' | 'warning' | 'info' | 'positive';
  priority: number;              // 1(최고) ~ 10(최저)
  title: string;                 // 한 줄 헤드라인 (≤ 30자)
  titleSimple: string;           // 비전문가용 헤드라인 (≤ 20자)
  description: string;           // 상세 설명 (1~2문장, 전문 용어 포함)
  descriptionSimple: string;     // 비전문가용 설명 (전문 용어 없이)
  relatedIndicator: string;      // 연관 지표 ID
  relatedChart?: string;         // 연관 차트 ID
  value: number | string;        // 근거 수치
  valueContext?: string;         // 수치 근거 설명 ("전월 대비", "벤치마크 대비" 등)
  insightDepth: 'descriptive' | 'diagnostic' | 'predictive' | 'prescriptive';
  icon: string;                  // 이모지 또는 아이콘 키
}
```

### 1-1a. 이중 언어 인사이트 규칙 (Dual-Tone)

모든 인사이트는 **전문가용**과 **일반 사용자용** 두 가지 톤으로 생성한다.
UI에서는 토글 또는 사용자 설정으로 전환 가능하며, 기본값은 `simple` 모드.

| 필드 | 전문가 모드 (Expert) | 일반 모드 (Simple) |
|------|---------------------|-------------------|
| `title` | "RSI 과매수 진입 (73.2)" | — |
| `titleSimple` | — | "매수 과열 신호" |
| `description` | "RSI 14일 기준 73.2로 과매수 구간(>70) 진입. 단기 조정 가능성." | — |
| `descriptionSimple` | — | "최근 많이 올라서 잠시 쉬어갈 수 있어요." |

**작성 규칙:**
- `titleSimple`: 전문 용어 금지, 동사형 문장 ("~하고 있어요", "~해요")
- `descriptionSimple`: 비유/일상 표현 사용, 수치는 "약 X%" 형태로 단순화
- `valueContext`: 비교 맥락을 항상 포함 (전월 대비, 벤치마크 대비, 52주 최고 대비 등)

### 1-2. 심각도 레벨 정의

| 레벨 | 배지 색상 | 아이콘 | 용도 |
|------|----------|--------|------|
| `alert` | 빨강 (`#EF4444`) | ⚠️ | 급격한 하락, MDD 경고, 극단적 집중 |
| `warning` | 주황 (`#F59E0B`) | 🔔 | 주의 필요 신호 (과매수/과매도, 변동성 증가) |
| `info` | 파랑 (`#3B82F6`) | ℹ️ | 중립적 사실 전달 (추세, 통계 요약) |
| `positive` | 초록 (`#10B981`) | ✅ | 긍정적 신호 (안정적 수익, 분산 양호) |

### 1-3. 표시 위치

| 위치 | 설명 |
|------|------|
| `TOP_BANNER` | 대시보드 최상단 배너 (alert/positive 레벨만) |
| `KPI_CARD` | KPI 요약 카드 하단 부제 |
| `CHART_ANNOTATION` | 특정 차트 위 툴팁/마커 |
| `INSIGHT_PANEL` | 우측 인사이트 패널 (전체 목록) |

## 5. 시장 급락 필터 (Market Crash Filter)

ETF_COMP에서 복수 종목의 급락을 동시 감지하여 시장 전반의 위기를 판별:

```
marketCrashDetected = false

IF dataType === 'ETF_COMP':
    negativeCount = 종목 중 최근 2일 누적수익률 < config.marketCrashThreshold 인 종목 수
    IF negativeCount / 전체종목수 >= 0.7:    // 70% 이상 동시 급락
        marketCrashDetected = true
```

### InsightConfig 추가 필드
```typescript
  marketCrashThreshold: number;  // 기본 -5% — 2일 누적 수익률 기준
```

### 시장 급락 시 인사이트 규칙

| 조건 | 레벨 | 우선순위 | 전문 | 일반 |
|------|------|---------|------|------|
| marketCrashDetected=true | `alert` | 0 | "시장 급락 감지: {negativeCount}/{total} 종목이 2일간 {threshold}% 이상 하락" | "시장 전체가 크게 빠지고 있어요! 조심해야 할 시점이에요" |

### 시장 급락 시 행동 규칙
- 요약 배너: `alert` 레벨로 강제 전환, 빨간 배경
- 기존 `positive` 레벨 인사이트 모두 일시 억제 (혼란 방지)
- KPI 카드: 수익률 카드에 "⚠ 시장 급락 구간" 부제 강제 표시

## 5a. 서사적 일관성 검증 (Narrative Consistency) — PORT_ALLOC 전용

투자자의 포트폴리오 구성이 데이터에서 드러나는 투자 성향과 일관되는지 검증:

```
IF dataType === 'PORT_ALLOC':
    // 1. 포트폴리오 성향 판별
    avgVolatility = 가중평균 변동성
    topAssetWeight = 최대 비중 자산의 비중
    
    // 2. 성향 분류
    IF avgVolatility < 15% AND topAssetWeight < 30%:
        profileLabel = "안정-분산형"
    ELSE IF avgVolatility > 30% OR topAssetWeight > 50%:
        profileLabel = "공격-집중형"
    ELSE:
        profileLabel = "균형형"
    
    // 3. 불일치 감지
    inconsistencies = []
    IF profileLabel === "안정-분산형" AND 개별자산 중 변동성>40% 존재:
        inconsistencies.push("안정형 포트폴리오에 고변동성 자산 포함")
    IF profileLabel === "공격-집중형" AND HHI < 0.10:
        inconsistencies.push("공격형 표방이지만 과도하게 분산")
```

| 조건 | 레벨 | 우선순위 | 전문 | 일반 |
|------|------|---------|------|------|
| inconsistencies 존재 | `warning` | 2 | "포트폴리오 성향 불일치: {profileLabel}이나 {inconsistency}" | "포트폴리오 구성이 투자 성향과 안 맞을 수 있어요" |
| inconsistencies 없음 | `positive` | 6 | "포트폴리오 성향 일관성 양호 ({profileLabel})" | "투자 성향에 맞게 잘 구성되어 있어요!" |

## 6. 인사이트 정렬 및 표시 규칙

### 4단계 인사이트 프레임워크 (Insight Depth Levels)

각 인사이트는 분석 깊이에 따라 다음 4단계 중 하나에 해당하며, `insightDepth` 필드로 표기:

| 단계 | 이름 | 설명 | 예시 |
|------|------|------|------|
| 1 | `descriptive` | 현상 파악 — "무엇이 일어났는가" | "총수익률 -12.3%, 하락 추세" |
| 2 | `diagnostic` | 원인 분석 — "왜 일어났는가" | "IT 섹터(-18%)가 전체 하락 견인" |
| 3 | `predictive` | 예측 — "어떻게 될 것인가" | "현재 추세 지속 시 30일 후 MDD -20% 예상" |
| 4 | `prescriptive` | 행동 제안 — "어떻게 해야 하는가" | "HHI 0.32로 집중도 높음, 리밸런싱 고려" |

**적용 규칙:**
- 현재 인사이트 규칙(§2~§4)의 기존 인사이트는 대부분 `descriptive` 또는 `diagnostic`
- `prescriptive` 인사이트: 포트폴리오 관련(HHI, 비중 경고, 서사적 일관성)에만 적용
- `predictive`는 v2.1에서는 미구현 — 향후 시계열 예측 모델 연동 시 확장 가능
- `insightDepth` 필드는 Insight 인터페이스에 추가

### 정렬 기준
1. **레벨 우선**: alert > warning > positive > info
2. **동일 레벨 내**: priority 숫자 오름차순 (1이 최우선)
3. **동일 priority**: 지표 값의 절대 크기 내림차순

### 표시 제한
| 영역 | 최대 표시 수 |
|------|-------------|
| `TOP_BANNER` | 1개 (가장 높은 우선순위) |
| `KPI_CARD` | 카드당 1개 |
| `CHART_ANNOTATION` | 차트당 2개 |
| `INSIGHT_PANEL` | 최대 10개 (스크롤) |

### 중복 방지
- 동일 지표에서 발생한 인사이트가 2개 이상이면, 가장 높은 priority만 표시
- 상반되는 인사이트(예: "상승 추세" + "데드크로스")가 동시 발생하면 둘 다 표시하되, 더 최신 데이터 기반 인사이트에 `(최신)` 태그 부여

### Null-Safety 필터링 (필수)
인사이트 배열을 최종 출력하기 전 반드시 필터링:

```
insights = insights.filter(insight =>
  insight.value !== null &&
  insight.value !== undefined &&
  insight.relatedIndicator !== null &&
  !isNaN(insight.value)    // 수치 기반 인사이트의 NaN 방지
);
```

추가 규칙:
- 지표 계산이 실패(null)된 경우, 해당 지표를 참조하는 인사이트는 **자동 제외**
- DetectionResult의 `confidence < 0.5`인 경우, info 레벨 인사이트만 생성 (alert/warning 억제)
- `fallbackMode: true`인 경우, "분석 가능한 투자 지표가 부족합니다" info 인사이트 1개만 생성

### 이상치 경고 인사이트
지표계산규칙의 "이상치 감지(Anomaly Guard)"에서 경고가 발생한 경우:

| 이상치 유형 | 레벨 | 템플릿 |
|------------|------|--------|
| 액면분할/병합 의심 | `warning` | "{date}에 {value}% 급변 감지, 액면분할·병합 가능성" |
| 가격 0/음수 | `alert` | "데이터에 비정상 가격(0 또는 음수)이 {count}건 포함" |
| 거래 정지 의심 | `info` | "{start}~{end} 구간 {days}일간 가격 변동 없음" |

## 8. 설정 가능한 임계값 (Configurable Thresholds)

관리자(P2)가 분석 기준을 환경에 맞게 조정할 수 있도록, 인사이트 규칙의 주요 수치를 파라미터로 분리한다. 기본값은 즉시 사용 가능하며, 오버라이드 시 대시보드 설정 패널에서 변경한다.

```typescript
interface InsightConfig {
  // 수익률 임계값
  returnThresholds: {
    strongPositive: number;   // 기본 +20% — 이상 → positive
    weakNegative: number;     // 기본 -10% — 이상 → warning
    strongNegative: number;   // 기본 -10% — 미만 → alert
  };
  // 변동성 임계값
  volatilityThresholds: {
    high: number;             // 기본 40% — 이상 → alert
    elevated: number;         // 기본 25% — 이상 → warning
    low: number;              // 기본 15% — 미만 → positive
  };
  // MDD 임계값
  mddThresholds: {
    severe: number;           // 기본 -30% — 미만 → alert
    moderate: number;         // 기본 -15% — 미만 → warning
    mild: number;             // 기본 -5%  — 이상 → positive
  };
  // RSI 임계값
  rsiThresholds: {
    overbought: number;       // 기본 70
    oversold: number;         // 기본 30
  };
  // 포트폴리오 집중도
  concentrationThresholds: {
    singleAssetMax: number;   // 기본 50% — 단일 자산 비중 경고
    categoryMax: number;      // 기본 60% — 단일 카테고리 비중 경고
    hhiHigh: number;          // 기본 0.25
    hhiLow: number;           // 기본 0.15
  };
  // 무위험 수익률
  riskFreeRate: number;       // 기본 3.5% (연)
  // 시장 급락 필터
  marketCrashThreshold: number; // 기본 -5% — 2일 누적 수익률 기준 (ETF_COMP 전용)
  // 인사이트 표시 모드
  displayMode: 'simple' | 'expert';  // 기본 'simple'
}
```

**규칙**: 인사이트 조건문에서 하드코딩된 수치를 사용하지 않고, 항상 `config.xxxThresholds` 값을 참조. 이를 통해 한국 시장(변동성 낮음) vs 미국 시장(변동성 높음) 등 환경별 맞춤 가능.

## 9. 요약 배너 규칙 (Summary Banner)

대시보드 최상단에 **1문장 결론**을 반드시 표시. 임원·개인 투자자가 3초 만에 핵심을 파악하도록 한다.

### 생성 규칙
```
가장 높은 priority의 인사이트를 기반으로 요약문 생성:
  - alert 존재 → "⚠️ {titleSimple} — {descriptionSimple}"
  - warning만 → "🔔 {titleSimple} — {descriptionSimple}"  
  - positive만 → "✅ {titleSimple} — {descriptionSimple}"
  - info만 → "ℹ️ 현재 특이사항 없이 안정적입니다."
```

### displayMode 전환 규칙
```
IF config.displayMode === 'simple':
    topBanner.summaryText = `${insight.titleSimple} — ${insight.descriptionSimple}`
    kpiCards[].subtitle = insight.titleSimple
    summary = summarySimple
ELSE:  // 'expert'
    topBanner.summaryText = `${insight.title} — ${insight.description}`
    kpiCards[].subtitle = insight.title
    summary = summaryExpert
```

### 배너 디자인 규칙
| 레벨 | 배경색 | 텍스트 색 |
|------|--------|----------|
| alert | `#FEF2F2` (연빨강) | `#991B1B` |
| warning | `#FFFBEB` (연주황) | `#92400E` |
| positive | `#ECFDF5` (연초록) | `#065F46` |
| info | `#EFF6FF` (연파랑) | `#1E40AF` |

## 10. 드릴다운 상세 뷰 규칙

각 차트 카드는 클릭 시 **상세 뷰**로 전환 가능:

### 상세 뷰 구성
| 영역 | 내용 |
|------|------|
| 확대 차트 | 전체 너비, 인터랙티브 (줌, 팬, 범례 토글) |
| 수치 근거 | 해당 지표의 계산 과정 요약 (공식 + 입력값 + 결과) |
| 관련 인사이트 | 이 차트와 relatedChart로 연결된 인사이트 목록 |
| 원본 데이터 | 해당 지표 계산에 사용된 데이터 테이블 (최대 100행) |

### 드릴다운 진입 규칙
- 모든 FULL/HALF 크기 차트 카드에 드릴다운 아이콘(🔍) 표시
- QUARTER 크기(KPI 카드)는 드릴다운 불가 (공간 부족)
- fallbackMode 시 드릴다운 비활성화

## 11. 출력 스키마

```typescript
interface InsightResult {
  config: InsightConfig;       // 적용된 설정값
  insights: Insight[];

  summary: string;             // 전체 요약 (2~3문장, simple 모드)
```

## 12. 초보자 친화 메시지 변환 템플릿 (v2.2 신규)

> 복잡한 수식이나 전문 용어를 **일상어 메시지**로 변환하는 규칙.
> `displayMode === 'simple'` 또는 페르소나 P1(입문자), P3(장기투자자)일 때 우선 적용.
> 메시지는 "숫자 → 의미" 순서로 전달한다.

---

### 12-1. 괴리율 메시지 변환

| 조건 | simple 메시지 | expert 메시지 |
|------|--------------|--------------|
| 괴리율 < -1% | "현재 이 ETF는 실제 가치(NAV)보다 **{n}% 저렴하게** 거래되고 있습니다. 저평가 구간일 수 있습니다." | "프리미엄/디스카운트: {n}% (괴리율 정상 범위 ±0.5% 초과)" |
| -1% ≤ 괴리율 ≤ 1% | "이 ETF는 실제 가치와 **거의 같은 가격**에 거래되고 있습니다." | "괴리율: {n}% (정상 범위 내)" |
| 괴리율 > 1% | "⚠️ 이 ETF는 실제 가치보다 **{n}% 비싸게** 거래 중입니다. 매수 타이밍에 주의하세요." | "프리미엄: +{n}% (과열 주의, 정상 범위 ±0.5% 초과)" |
| 괴리율 > 3% | "🚨 이 ETF는 실제 가치보다 **{n}% 비싸게** 거래되고 있어 위험 수준입니다. 신중한 접근이 필요합니다." | "프리미엄: +{n}% (위험 수준, 즉각 점검 필요)" |

---

### 12-2. MDD(최대 낙폭) 메시지 변환

| 조건 | simple 메시지 | expert 메시지 |
|------|--------------|--------------|
| MDD > -10% | "최근 고점 이후 가장 크게 떨어진 폭이 **{n}%** 입니다. 상대적으로 안정적입니다." | "MDD: {n}% (저위험 구간)" |
| -20% < MDD ≤ -10% | "최근 고점 대비 최대 **{n}%** 하락한 구간이 있었습니다. 과거 유사 구간에서 회복까지 평균적으로 시간이 필요했습니다." | "MDD: {n}% (중위험, 모니터링 권고)" |
| -30% < MDD ≤ -20% | "⚠️ 최대 낙폭이 **{n}%** 로 상당한 손실 구간이 존재했습니다. 투자 전 손실 허용 범위를 확인하세요." | "MDD: {n}% (고위험, 리스크 관리 점검 필요)" |
| MDD ≤ -30% | "🚨 최대 낙폭이 **{n}%** 로 매우 심각합니다. 이 수준의 하락을 견딜 수 있는지 신중하게 검토하세요." | "MDD: {n}% (극고위험, 포지션 조정 검토)" |

### 12-3. 수익률 메시지 변환

| 조건 | simple 메시지 | expert 메시지 |
|------|--------------|--------------|
| 총수익률 > 20% | "선택 기간 동안 **{n}% 수익**을 올렸습니다. 우수한 성과입니다." | "총수익률: +{n}%, 연환산: {a}%" |
| 0% < 총수익률 ≤ 20% | "선택 기간 동안 **{n}% 수익**을 올렸습니다." | "총수익률: +{n}%, 연환산: {a}%" |
| -10% < 총수익률 ≤ 0% | "선택 기간 동안 **{n}% 손실**이 발생했습니다." | "총수익률: {n}%, 연환산: {a}%" |
| 총수익률 ≤ -10% | "⚠️ 선택 기간 동안 **{n}% 손실**이 발생했습니다. 원인 분석이 필요합니다." | "총수익률: {n}%, 연환산: {a}% (손실 구간)" |

### 12-4. 골든크로스/데드크로스 메시지 변환

| 조건 | simple 메시지 | expert 메시지 |

| 데드크로스 발생 | "단기 흐름이 장기 흐름을 **아래로 이탈**했습니다. 하락 추세 진입 가능성이 있습니다 (데드크로스)." | "데드크로스 감지: {날짜}, MA5 < MA20 (거래량 확인: {confirmed})" |

### 12-5. 포트폴리오 성향 불일치 메시지 변환

| 조건 | simple 메시지 | expert 메시지 |
|------|--------------|--------------|

| 공격추구 + 과분산 | "지나치게 많은 자산에 분산되어 **수익 집중력이 낮습니다**. 핵심 종목 비중 확대를 검토해 보세요." | "보유종목 수: {n}개 (공격-집중형 프로파일 대비 과분산)" |

### 12-6. 메시지 생성 우선순위

같은 지표에 여러 조건이 겹칠 때:
1. **alert** 메시지를 항상 먼저 표시
2. 동일 심각도 내에서는 **괴리율 > MDD > 수익률** 순
3. 1개 데이터 유형당 초보자 메시지는 **최대 3개** 표시 (초과 시 상위 심각도 우선)
4. 같은 지표의 expert/simple 메시지를 동시에 표시하지 않음
