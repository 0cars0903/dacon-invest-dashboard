# 🧑‍💼 페르소나 분석 — InvestLens

## P1. 개인 투자자 (Primary User)

| 항목 | 내용 |
|------|------|
| 역할 | 본인 자산을 직접 관리하는 개인 |
| 기술 수준 | 낮음~중간 (CSV 다운로드 가능, 코드 작성 불가) |
| 핵심 니즈 | "내 포트폴리오가 지금 괜찮은지" 한눈에 파악 |
| 데이터 소스 | 증권사 앱 CSV 내보내기, 엑셀 직접 작성 |
| 사용 빈도 | 주 1~2회 |
| 고통점 | 분석 용어가 어려움, 차트만 봐서는 판단 불가 |

**Skills.md 반영 포인트:**
- 인사이트 문구를 **평이한 한국어**로 (전문 용어 옆에 괄호 설명)
- KPI 카드에 **상승/하락 화살표 + 색상**으로 직관적 표현
- "지금 어떤 상태인가"를 1문장으로 보여주는 **요약 배너**

## P2. 관리자 / 설정자 (Admin)

| 항목 | 내용 |
|------|------|
| 역할 | 대시보드를 세팅하고 분석 기준을 조정하는 사람 |
| 기술 수준 | 중간~높음 (Skills.md 수정 가능) |
| 핵심 니즈 | 분석 규칙을 커스터마이즈, 팀원에게 대시보드 공유 |
| 데이터 소스 | API 연동, 대규모 CSV, 자체 데이터 파이프라인 |
| 사용 빈도 | 초기 설정 집중, 이후 월 1~2회 유지보수 |
| 고통점 | 새 데이터 형식이 들어올 때마다 규칙 수정 필요 |

**Skills.md 반영 포인트:**
- 데이터감지규칙의 **패턴 매칭 확장성** (커스텀 패턴 추가 섹션)
- 시각화매핑규칙에 **차트 우선순위 오버라이드** 기능
- 인사이트 임계값(예: MDD 경고 기준 -15% → -20%)을 **설정 가능한 파라미터**로

## P3. 데이터 분석가 (Analyst)

| 항목 | 내용 |
|------|------|
| 역할 | 투자 데이터를 깊이 분석하는 전문가 |
| 기술 수준 | 높음 (Python/R 능숙, 통계 지식 보유) |
| 핵심 니즈 | 빠른 EDA, 다양한 데이터셋 비교, 리포트 공유 |
| 데이터 소스 | Yahoo Finance, FRED, Bloomberg, 자체 DB |
| 사용 빈도 | 매일 |
| 고통점 | 매번 같은 분석 코드를 반복 작성하는 비효율 |

**Skills.md 반영 포인트:**
- 지표계산규칙에 **선택 지표 확장** (알파, 베타, 트레이너 비율 등)
- 시각화에 **상세 뷰(드릴다운)** 규칙 추가
- 인사이트에 **수치 근거 표시** (단순 문구가 아닌 계산 과정 힌트)

---

## P4. 퀀트 개발자 (Power User)

| 항목 | 내용 |
|------|------|
| 역할 | Skills.md 자체를 수정·확장하여 자동화 파이프라인 구축 |
| 기술 수준 | 매우 높음 |
| 핵심 니즈 | Skills.md의 모듈성, API/프로그래밍 인터페이스 |
| 고통점 | 규칙이 하드코딩되어 확장이 어려움 |

**Skills.md 반영 포인트:**
- 각 모듈의 **입출력 TypeScript 인터페이스** 명확화 (이미 반영됨 ✅)
- 지표계산규칙에 **커스텀 지표 플러그인** 구조 언급
- 시각화매핑규칙에 **커스텀 차트 등록** 가능성 언급

---

## P5. 임원 / 열람자 (Viewer)

| 항목 | 내용 |
|------|------|
| 역할 | 분석 결과만 열람, 의사결정에 활용 |
| 기술 수준 | 낮음 (데이터 업로드도 안 함) |
| 핵심 니즈 | 3초 안에 핵심 파악, 인쇄/공유 가능 |
| 사용 빈도 | 주 1회 리포트 확인 |
| 고통점 | 차트가 많으면 오히려 혼란, "결론이 뭔데?" |

**Skills.md 반영 포인트:**
- 인사이트 **요약(summary) 필드** 강화 (2~3문장 전체 요약)
- 시각화에 **프린트 최적화 레이아웃** 규칙
- KPI 카드를 **가장 먼저, 가장 크게** 표시

---

## P6. 자문사 / 리포트 생성자 (Advisor)

| 항목 | 내용 |
|------|------|
| 역할 | 고객에게 투자 분석 리포트를 제공하는 자문 역할 |
| 기술 수준 | 중간 |
| 핵심 니즈 | 고객별 맞춤 분석, 브랜딩된 리포트 출력 |
| 사용 빈도 | 주 3~5회 |
| 고통점 | 고객마다 데이터 형식이 다름, 일관된 리포트 어려움 |

**Skills.md 반영 포인트:**
- 데이터감지규칙의 **범용성**이 이 페르소나에 직결 (다양한 고객 데이터 대응)
- 인사이트에 **비교 맥락** 추가 ("벤치마크 대비", "전월 대비")
- 시각화에 **내보내기(PDF/이미지)** 기능 언급
